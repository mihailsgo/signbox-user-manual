# User and Access Management

This page explains what normal users should expect from administrators and what is usually outside normal user control.

## What Users Normally Ask Administrators For

Contact your administrator or support owner when you need:
- account creation or enablement
- password reset or access recovery
- role assignment changes
- country or signing-method availability clarification
- contact and template scope policy clarification
- help with tenant-specific process behavior

## What Is Usually Not Controlled by End Users

Normal users typically cannot change:
- identity provider configuration
- signing-method integrations
- archive and retention policy
- tenant-wide document-type mapping
- global contact or template policy
- recipient matching rules for regulated flows

## Role and Scope Basics

Common scope model:
- `Personal`: visible only to the creator
- `Group`: shared inside a defined user group
- `Global`: available broadly across the tenant
- `All`: filter value, not an ownership scope

What this affects:
- who can see contacts
- who can see templates
- who can reuse shared setup
- who can maintain shared records

## Questions to Confirm with Your Administrator

If your process is compliance-sensitive, confirm:
- whether anonymous recipient flow is allowed
- which process types require personal code matching
- whether initiators may update active processes
- whether deletion affects only UI history or also archived content
- which document types should be used for each business scenario

## Support Escalation Checklist

When escalating an issue, include:
- process ID
- timestamp and timezone
- user role
- exact action taken
- exact error text
- screenshot with sensitive content redacted

This reduces turnaround time significantly.

## Recommended Ownership Model

For a healthy operating model, organizations should know who owns:
- user access
- process profile definitions
- archive behavior
- external signing support
- first-line user support

If the owner is not obvious in your environment, ask for the correct internal support contact and keep it documented together with this manual.
