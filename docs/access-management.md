# User and Access Management 🔐

## What users should ask admins for
- Account enablement/reset
- Role assignment
- Available signing methods and countries
- Contact/template scope policy

## Scope policy quick matrix
- `Personal`: owner-only visibility and ownership.
- `Group`: shared inside one group; requires group membership.
- `Global`: tenant-wide visibility.
- `All`: filter helper used in UI/API queries.

## Anonymous policy questions to confirm with admin
- Is anonymous recipient flow allowed for your legal/compliance process type?
- Which process types require mandatory personal code matching?
- Is invitation-link-only recipient matching allowed in your tenant?

## What is not in normal user control
- Identity provider settings
- Method-provider integrations
- Global retention/deletion policies

## Escalation checklist
When reporting issue, include:
- Process ID
- Timestamp and timezone
- User role
- Exact action and error text
- Screenshot (redacted)
