# Terminology

Use these terms consistently in SignBox.

- **Process**: A signing workflow that contains document(s), recipients, status, and timeline.
- **Container**: The final signed package/file name produced by the process.
- **Recipient**: A participant in a process.
- **Recipient group**: One workflow step containing one or more recipients.
- **Role**:
  - `Signer`: signs document(s)
  - `Viewer`: views document(s)
  - `Approver`: approval role where configured
- **Anonymous recipient**: Recipient created without personal code/phone in process payload.
- **Non-anonymous recipient**: Recipient created with personal code/phone.
- **Due date**: Deadline for a recipient group step.
- **Template**: Saved process setup for reuse.
- **Contacts**: Saved recipient records.
- **History**: Process list and filters for tracking and management.

For full definitions, see [Glossary](glossary.md).
