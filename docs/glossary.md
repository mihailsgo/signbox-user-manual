# Glossary 📚

- **ASiC-E**: Container format used for signed packages.
- **Anonymous recipient**: Recipient record created without personal code in process payload.
- **Approver**: Recipient role used for approval step where configured.
- **Container name**: Name of final package/file generated by process.
- **Contacts**: Saved recipient entries for reuse.
- **Coverage report**: Mapping showing user questions are addressed by documentation sections.
- **Draft**: Unstarted process state.
- **Due date**: Deadline set for a recipient group step.
- **History**: Process list page with filters and statuses.
- **Initiator**: User who creates and launches a process.
- **Non-anonymous recipient**: Recipient created with personal identifier (personal code/phone).
- **Process**: Full workflow instance for document signing.
- **Recipient**: Person participating in process.
- **Recipient group**: One workflow step containing one or more recipients.
- **Sign first**: Option for initiator-first signing behavior.
- **Signer**: Recipient role that performs signing.
- **Template**: Saved reusable process setup.
- **Viewer**: Recipient role that can view process/documents.
