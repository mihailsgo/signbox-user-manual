# Overview 🧭

SignBox has two main portals:

- **Internal portal** (`signbox.<tenant>`): create, manage, and track processes.
- **External portal** (`esign.<tenant>`): recipient signing experience.

## Typical Use Cases
- Employee/HR contract signing
- Internal approval chains
- Multi-recipient document workflows

## User Roles
- **Initiator**: creates process, sets recipients, tracks status
- **Recipient**: receives invitation, signs or declines
- **Admin** (if assigned): advanced management sections

> [!WARNING]
> Treat all process links as sensitive. Do not forward recipient links to unauthorized people.
